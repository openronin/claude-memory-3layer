# claude-memory-3layer

> Ручная, хранящаяся в репозитории память для Claude Code. Три уровня, осознанное размещение, без автосбора. С гибридным поиском (qmd) и картой символов по требованию (ctags). Всё локально.

Замена стандартной системы памяти Claude Code, проверенная на ~15 реальных проектах (PHP/mpcmf, Python, TS, реверс-инжиниринг, мультимашинные установки).

## Зачем это нужно

После оценки популярных альтернатив:
- **[CodeGraph](https://github.com/colbymchenry/codegraph)** — граф знаний о коде. На момент оценки содержал живой баг в MCP-транспорте; баг корректности при поиске символов с квалификатором модуля (тихо выбирает случайные коллизии).
- **[AgentMemory](https://github.com/rohitg00/agentmemory)** — постоянная память через MCP. Issue #522: *"Сессии никогда не создаются, данные теряются без предупреждения"*. Философски противоположно тому, что нужно агенту для написания кода (автосбор в векторный блоб против явного, курируемого знания).

Эта система — то, что сформировалось как альтернатива: **что ты записал — твоё. Что не записал — не существует.** Никаких векторных блобов, автосбора, демонов, MCP-серверов. Только markdown и минимальный набор инструментов.

## Ключевой принцип

> *«Понадобилось бы это серверному агенту в полночь, без памяти и без человека?»*
> — **да** → кладём в репозиторий (git-tracked, путешествует вместе с кодом)
> — **нет** → кладём в локальную память аккаунта (`~/.claude/...`)

Факты о кодовой базе живут рядом с кодом. Личные предпочтения и рабочие заметки — локально в аккаунте. Не смешивай.

## Архитектура — три уровня с разным временем жизни

| Уровень | Путь | Git? | Время жизни |
|---|---|---|---|
| **L0 Личность** | `~/.claude/memory/IDENTITY.md` | нет | постоянно (≤25 строк) |
| **L1a Вход в репозиторий** | `<repo>/CLAUDE.md` | **да** | на проект; тонкий, автозагружается |
| **L1b Документы репозитория** | `<repo>/.claude-docs/*.md` | **да** | на проект; объёмный, загружается по требованию |
| **L1 запасной** | `~/.claude/projects/<slug>/memory/project.md` | нет | для репозиториев, где вариант «в репозитории» неуместен |
| **L2 Сессия** | `~/.claude/projects/<slug>/memory/SESSION.md` | нет | на задачу; умирает, когда задача завершена |

`<repo>/CLAUDE.md` загружается Claude Code автоматически в каждую сессию — держи его **тонким**. Объёмные знания живут в `<repo>/.claude-docs/*.md` и читаются по требованию через индекс в `CLAUDE.md`.

Главный ход — **`<repo>/.claude-docs/gotchas.md`** — каждый неочевидный подводный камень попадает сюда. *«Выглядит правильно, но ломается»* / *«Выглядит неправильно, но так задумано»*. По одному абзацу на каждый. Будущие сессии натыкаются на ту же стену — и сразу видят предупреждение.

## Как это работает — пошагово

### 1. Старт сессии

Хук **SessionStart** срабатывает при каждом открытии Claude Code:
1. Агент читает `~/.claude/memory/IDENTITY.md` (L0) — кто ты, твои предпочтения
2. `<repo>/CLAUDE.md` загружается автоматически самим Claude Code — команды, соглашения, индекс документации
3. Агент читает `SESSION.md` — рабочее состояние с прошлого раза (если существует)
4. Если `SESSION.md` не обновлялся >24ч — хук **предупреждает об устаревании**, агент явно спрашивает: «продолжить или сбросить?»

### 2. Во время работы

- Агент пишет в `SESSION.md` после каждого решения: что сделано, что дальше, карта ключевых файлов
- Обнаружил подводный камень → добавил в `<repo>/.claude-docs/gotchas.md`, ты коммитишь это как часть работы
- Объёмные знания о проекте → в `<repo>/.claude-docs/architecture.md`, `conventions.md`, и т.д.

### 3. Перед компакцией

Когда контекст заполняется и нужна компакция:
1. Скажи «теперь можно компакт» / «compact me»
2. Агент делает **PRE-COMPACT CHECKPOINT**: проверяет git-статус, сбрасывает в `SESSION.md` всё рабочее состояние (включая дословные цитаты последних сообщений)
3. После компакции — первое действие агента: читает `SESSION.md` обратно

### 4. Завершение задачи

- Без «запомни»: `SESSION.md` сбрасывается до пустого шаблона. Автоархивирования нет.
- С «запомни»: нужная информация переходит в `IDENTITY.md` (личное) или в `<repo>/.claude-docs/<файл>.md` (факт о кодовой базе, ты коммитишь)

## Структура уровней подробно

### L0 — Личность (`~/.claude/memory/IDENTITY.md`)

- **Что хранит:** кто ты, язык, ОС/шелл, жёсткие предпочтения, учётные данные уровня окружения (SSH-обёртки, AWS-профили)
- **Жёсткий лимит: 25 строк** — загружается в каждую сессию всех проектов
- **Не трогай:** сюда не идут факты о конкретных проектах

### L1a — Вход в репозиторий (`<repo>/CLAUDE.md`)

- **Что хранит:** команды сборки/тестов/линтера, соглашения верхнего уровня, ОБЯЗАТЕЛЬНО/НЕЛЬЗЯ, индекс документации
- **Держи тонким:** загружается в каждую сессию в этом репозитории — раздутый файл ест контекст
- **Git-tracked:** путешествует с кодом, видят все (коллеги, серверные агенты)

### L1b — Документы репозитория (`<repo>/.claude-docs/*.md`)

- **Что хранит:** архитектура, соглашения, паттерны, рецепты, **и главное — `gotchas.md`**
- **Загружается по требованию:** агент читает нужный файл только когда задача требует этой информации
- **Git-tracked:** версионируется вместе с кодом

**Файлы L1b:**
- `index.md` — таблица маршрутизации: «нужно сделать X → читай Y.md»
- `gotchas.md` — неочевидные подводные камни (высочайшая ценность!)
- `architecture.md` — стек, структура директорий, мидлвары
- `conventions.md` — стиль кода, именование, формат коммитов
- `patterns.md`, `cookbook.md`, `api.md` — добавляй по мере необходимости

### L1 запасной (`~/.claude/projects/<slug>/memory/project.md`)

- **Когда использовать:** проект без репозитория, или где in-repo L1 неуместен (проприетарный, соло-эксперименты)
- **Slug** вычисляется из cwd: `C:\dev\foo` → `C--dev-foo`

### L2 — Сессия (`~/.claude/projects/<slug>/memory/SESSION.md`)

- **Что хранит:** цель текущей задачи, статус, принятые решения с обоснованием, карта файлов, открытые вопросы, дословные последние сообщения
- **Обновляется непрерывно** во время работы, не в конце
- **Умирает** при завершении задачи (без явного «запомни»)
- **Выживает** при компакции — единственный артефакт с полной точностью после сжатия контекста

## Инструменты (все локальные, без демонов)

### `/recall <запрос>` — гибридный поиск по памяти

BM25 + GGUF-эмбеддинги + LLM-ранжировщик по всем файлам памяти. На основе [qmd](https://github.com/tobi/qmd) (Tobi Lütke).

```
/recall MongoDB legacy driver mpcmf
/recall гочи WSL DNS IPv6
/recall --here armenia weekly window     # ограничить текущим проектом
```

### `/codemap def|callers|callees|outline <символ>` — карта символов по требованию

universal-ctags + ripgrep. Кэш в `<repo>/.codemap.tags` (можно добавить в .gitignore). Автоперестройка при изменении исходников.

```
/codemap def LlmClient::chat
/codemap callers AgentExecutor
/codemap outline
```

### `/memory status | auto on|off | refresh` — управление протоколом

По умолчанию режим **явного переноса** — кросс-сессионная память только когда пользователь говорит «запомни». Переключай на автозахват на сессию при необходимости.

### `/memstat [--watch]` — мониторинг подсистемы памяти

Показывает запущенные процессы qmd/ctags (PID, RAM, время), прогресс индексации (векторов обработано / ожидает, % покрытия), расписание обновлений, последние записи в логе. Используй когда `node.exe` ест CPU и непонятно что происходит.

## Хуки (поддерживают дисциплину)

- **SessionStart** — инжектирует напоминание о протоколе; обнаруживает устаревший `SESSION.md` (>24ч с момента обновления → принудительный явный вопрос пользователю «продолжить или сбросить?»). Также запускает в фоне `qmd update + embed` с дедупликацией раз в 6ч.
- **PreCompact** — напоминает агенту сбросить рабочее состояние в `SESSION.md` перед тем как компакция сотрёт контекст. `SESSION.md` — единственный артефакт, выживающий при компакции с полной точностью.

## Чего намеренно нет в системе

- ✗ **Автозахват** произвольного вывода инструментов в память — именно такой режим отказа документируют в открытых issues `AgentMemory` (тихая потеря данных, разрастание логов).
- ✗ **MCP-сервер с 50+ инструментами** — налог на контекстное окно в каждом запросе, используешь их или нет.
- ✗ **Постоянный демон графа кода** с per-project SQLite — вместо него `/codemap` по требованию. Пересканирование за ~1с для средних репозиториев.
- ✗ **Векторный блоб**, который нельзя `diff`-нуть. Память — это markdown, который можно прочитать глазами.
- ✗ **Облако, ключи OpenAI, embedding API** — qmd использует локальные GGUF-модели (embeddinggemma-300M, qwen3-reranker, qmd-query-expansion).

## Быстрый старт

Полные инструкции для Windows / macOS / Linux — в [INSTALL.md](INSTALL.md). Кратко:

```bash
# 1. Помести CLAUDE.md, hooks, commands, bin/, templates в ~/.claude/
# 2. Смерджи settings.snippet.json в ~/.claude/settings.json
# 3. Отредактируй ~/.claude/memory/IDENTITY.md (≤25 строк, кто ты)
# 4. Для инструментов поиска (опционально, рекомендуется):
winget install OpenJS.NodeJS.LTS UniversalCtags.Ctags BurntSushi.ripgrep.MSVC
npm install -g @tobilu/qmd
qmd collection add ~/.claude/memory --name claude-l0
qmd collection add ~/.claude/projects --name claude-projects
QMD_LLAMA_GPU=none qmd embed   # один раз, скачает ~2GB GGUF-моделей
```

### Начало работы с проектом

Скажи агенту: *«Заведи L1 для этого проекта»* — он создаст файлы сам, заполнит что сможет вывести из кода, оставит заглушки для остального.

Или вручную:
```bash
cd /path/to/your/repo
mkdir -p .claude-docs
# скопируй шаблоны из templates/repo/ в проект
$EDITOR CLAUDE.md           # заполни команды, соглашения, ОБЯЗАТЕЛЬНО/НЕЛЬЗЯ
$EDITOR .claude-docs/architecture.md   # опиши стек и структуру
# gotchas.md оставь пустым — заполняй по мере обнаружения проблем
git add CLAUDE.md .claude-docs/
git commit -m "docs: add in-repo L1 memory scaffolding"
```

**Почему это важно:** знание путешествует с кодом. Клонируй репозиторий на новой машине → все gotchas, соглашения, архитектурные заметки уже там. Коллеги их видят. Серверные агенты (headless запуски Claude) их видят. Они версионируются вместе с кодом.

## Совместимость с Obsidian

Все файлы памяти используют YAML frontmatter с иерархическими тегами (`memory/l0` / `memory/l1` / `memory/l2` / `memory/repo`). Открой `~/.claude/` или корень любого репозитория как Obsidian vault — получишь граф-вью, фильтрацию по тегам, полнотекстовый поиск по всем проектам.

**Варианты vault:**
- **`~/.claude/` как vault** — кросс-проектный граф: L0 + все L2 сессии + L1-fallback project.md файлы. Полезно для «что я делал на прошлой неделе?»
- **`<repo>/` как vault** — погружение в один проект: `CLAUDE.md` + `.claude-docs/*` + твой код. Полезно для «покажи gotchas и архитектуру этой кодовой базы»

## Структура репозитория

```
CLAUDE.md                       — протокол памяти (заменяет стандартный; идёт в ~/.claude/)
INSTALL.md                      — инструкции по установке, решение проблем
LICENSE                         — MIT
CHANGELOG.md                    — история версий
memory/IDENTITY.md              — шаблон L0
templates/repo/CLAUDE.md        — шаблон L1a (тонкая точка входа в репозитории)
templates/repo/.claude-docs/*   — шаблоны L1b (gotchas, architecture, conventions, index)
templates/project.md.fallback.template — шаблон L1-fallback (локальная память аккаунта)
hooks/session-start.sh          — проверка устаревания + инжект протокола + автообновление qmd
hooks/pre-compact.sh            — напоминание о сбросе перед компакцией
commands/{recall,codemap,memory}.md — определения slash-команд
bin/codemap.sh                  — карта символов через universal-ctags + ripgrep
settings.snippet.json           — блок hooks для ~/.claude/settings.json
```
